1.
Create a fresh, professional folder
Instead of naming it py, let's give it a more professional name for your interview prep, l
like django_projects. Use the mkdir (make directory) command:

mkdir django_projects
cd django_projects

2.
you're now in a clean directory at /Users/macuser/django_projects. 
This is the professional way to start.
Since you are in a fresh folder, you need to rebuild the "engine" 
(the virtual environment) and get Django ready. Follow these steps exactly to keep it "interview-clean":

python3 -m venv .venv


3. Activate it.

source .venv/bin/activate

4. Install Django

pip install django

5.
. Create the Project (The "Pro" Way)

django-admin startproject my_website .
